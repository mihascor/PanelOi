## PanelOi — контекст проекта

Локальное десктоп-приложение (Python + PyQt6) для анализа данных MOEX.

Корень:
C:\Users\PC\PanelOi

---

## Назначение

- сбор рыночных данных (MOEX API)
- хранение в SQLite
- анализ открытого интереса (OI)
- визуализация в GUI

---

## Поток данных

MOEX API
↓
api (получение данных)
↓
services (обработка)
↓
repository (запись/чтение)
↓
storage (SQLite)
↓
services (подготовка данных)
↓
ui (отображение)

---

## Основные сущности

- instrument
- price
- oi
- signal

---

## Основной сценарий

1. sync_oi → загрузка данных
2. insert_oi → запись в БД
3. calculate_signals → расчёт
4. load_signal_to_ui → отображение

## База данных

storage/db/PanelOi_db.db

futures_list - хранилище тикетов фьючи и акции (ручная сборка)
CREATE TABLE "futures_list" (
        "futures"       TEXT,
        "paper" TEXT
);
sqlite>

futures_oi - исторические значения открытого интереса разбитые на - даты, инструмент,  
лица (юридические и физические), покупки и продажи.
CREATE TABLE futures_oi (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    futures TEXT,          -- код из futures_list
    paper TEXT,            -- человекочитаемое имя

    tradedate DATE,        -- дата

    FIZ_long INTEGER,
    FIZ_short INTEGER,

    YUR_long INTEGER,
    YUR_short INTEGER,

    UNIQUE(futures, tradedate)  -- защита от дублей
);

paper_price - исторические данные цен акций
CREATE TABLE "paper_price" (
        "id"    INTEGER,
        "ticker"        TEXT,
        "date"  DATE,
        "open"  REAL,
        "close" REAL,
        "high"  REAL,
        "low"   REAL,
        "volume"        INTEGER,
        PRIMARY KEY("id" AUTOINCREMENT)
);
CREATE UNIQUE INDEX idx_unique_ticker_date
ON paper_price (ticker, date);
