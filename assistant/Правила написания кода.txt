## Архитектура

Слои:

UI (PyQt)
↓
Services (бизнес-логика)
↓
Repository (работа с БД)
↓
Storage (SQLite)

Внешние источники:
- MOEX API

---

## Зависимости

- UI → services
- services → repository, api
- repository → storage
- api → внешний HTTP

Каждый слой зависит только вниз.

---

## Правила

1. UI не содержит бизнес-логики
2. Services не содержат SQL
3. Repository не знает про UI
4. API не работает с БД
5. доступ к БД только через repository
6. services формируют данные для UI

---

## Запрещено

- SQL в services
- API вызовы в UI
- бизнес-логика в UI
- прямой доступ к БД вне repository
- дублирование логики

---

## Правила работы с кодом (для ИИ) СТРОГО ВЫПОлНЯТЬ

- вносить точечные изменения
- не переписывать код без необходимости
- использовать формат:

  БЫЛО / СТАЛО + якорь строки

- по запросу писать файл целиком
- код комментировать на русском (ОБЯЗАТЕЛЬНО)
- импорты всегда сверху
- пути всегда относительные

## Логирование (обязательно)

1. Использовать только loguru
2. Инициализация логгера:
   - только в main.py через setup_logger()
   - запрещено настраивать логгер в других файлах

3. Локация логов:
   - storage/logs/app.log
   - storage/logs/error.log

4. Где логировать:
   - services → да
   - api → да
   - repository → только ошибки БД
   - ui → запрещено

5. Формат логов:
   - русский язык
   - обязательно:
     дата + время + уровень + источник + сообщение

6. Уровни:
   - logger.info → обычные операции
   - logger.success → успешное завершение
   - logger.warning → подозрительные ситуации
   - logger.error → ошибки

7. Запрещено:
   - использовать print
   - логировать UI-события (клики, кнопки)
   - дублировать логи
   - писать SQL или бизнес-данные напрямую в лог

8. Ротация логов:
   - обязательна
   - задаётся только в logger_config.py

Примеры:

- logger.info
- logger.success
- logger.warning
- logger.error

## UI Navigation

1. Навигация строится через Sidebar + Pages

2. Sidebar:
   - содержит только кнопки
   - не содержит логики
   - не знает про реализацию страниц

3. Pages:
   - каждая страница = отдельный файл
   - отвечает за один раздел (oi, price, logs, analytics)
   - может содержать свою локальную UI-логику (кнопки, layout)

4. MainWindow:
   - управляет переключением страниц
   - содержит QStackedWidget
   - связывает Sidebar и Pages

5. Запрещено:
   - использовать tabs как основной способ навигации
   - размещать глобальные кнопки вне страниц
   - смешивать несколько сущностей в одной странице

6. Допускается:
   - локальные панели внутри страниц (левая панель действий)
   - вложенные уровни внутри страницы
